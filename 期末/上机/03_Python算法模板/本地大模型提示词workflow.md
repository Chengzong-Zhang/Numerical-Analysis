# 数值方法上机题：本地9B模型提示词Workflow

> 适用场景：用本地9B模型（如Qwen、DeepSeek）写数值方法Python代码
> 核心原则：**任务拆得越细，模型越不容易出错**

---

## 一、拿到题目后的第一步：拆解问题

**不要直接让模型写代码。** 先发这条提示词：

```
我有一道数值方法题，请用中文回答以下问题，不要写任何代码：

题目：[粘贴题目]

请告诉我：
1. 这道题需要实现哪几个函数？（列出函数名和一句话描述）
2. 函数之间的调用顺序是什么？
3. 核心递归/迭代的终止条件是什么？
4. 误差是怎么估计的？用什么公式？
```

**目的**：让模型先建立正确的算法结构，暴露它理解中的错误，而不是把错误藏在代码里。

---

## 二、逐函数实现（最关键的步骤）

确认结构正确后，**一次只让它写一个函数**，并附上严格约束。

### 通用提示词模板

```
现在只写函数 [函数名]，其他函数先不写。

要求：
- 输入参数：[列出参数和类型]
- 输出：[描述返回值]
- 实现细节：[用伪代码或文字描述算法步骤]
- 禁止在函数内部定义嵌套函数
- 禁止使用任何尚未定义的函数
- 节点/系数等常量必须定义为列表或数组，不能是标量

只输出这一个函数的代码，不要写测试代码。
```

### 针对递归函数的额外约束

```
这个函数是递归的，额外要求：
- 必须先写终止条件（if 误差 < tol: return ...）
- 终止条件必须在递归调用之前
- 禁止在检查误差之前就调用递归
```

### 针对数值积分类的额外约束

```
关于节点和权重：
- 必须定义为列表，例如 nodes = [...], weights = [...]
- 禁止定义为单个标量
- 映射公式：x = (b-a)/2 * t + (a+b)/2，其中 t 是标准节点
```

## 三、组装与验证

所有函数写完后，发这条提示词：

```
请将以下函数组装成完整程序，并添加验证代码：

[粘贴已写好的各个函数]

验证部分要求：
- 定义题目给出的测试函数 f(x)
- 调用主算法得到结果
- 用高精度参考值（将区间分成2048份，每份用基础公式计算）作为对比
- 打印：计算结果、参考值、绝对误差
- 判断误差是否满足题目要求并打印结论
```

---

## 四、常见题型的关键约束速查

### 数值积分（自适应求积）

| 容易出错的地方 | 在提示词中加入的约束 |
|---|---|
| 节点定义为标量 | "节点必须是列表，禁止定义为标量" |
| 先递归再判断 | "先判断误差，满足条件才return，不满足才递归" |
| Richardson因子写错 | "误差估计公式为 abs(I_fine - I_coarse) / (2^(2n)-1)，两点GL用15" |
| 容差分配错误 | "递归调用时，左右各分配 tol/2，不是 tol/4" |

### 常微分方程（ODE）

| 容易出错的地方 | 在提示词中加入的约束 |
|---|---|
| 步长更新顺序 | "必须先计算k1,k2,...，全部计算完再更新y" |
| 自适应步长 | "先计算误差估计，再决定是否接受当前步，接受后才推进t" |
| 结果存储 | "每步的t和y都必须append到列表里，最后返回列表" |

### 最佳逼近多项式

| 容易出错的地方 | 在提示词中加入的约束 |
|---|---|
| 内积计算 | "内积 (f,g) 用数值积分实现，调用已写好的积分函数" |
| 法方程组装 | "Gram矩阵的第(i,j)项是基函数phi_i和phi_j的内积" |
| 基函数选取 | "基函数使用Legendre多项式，先写出前n+1个的表达式再实现" |

### 插值与样条

| 容易出错的地方 | 在提示词中加入的约束 |
|---|---|
| 差商表 | "差商表用二维列表实现，逐列填写，第0列是函数值" |
| 三弯矩方程 | "系数矩阵是三对角矩阵，用追赶法求解" |

## 五、出错后的修复提示词

如果运行报错，不要让模型重写整个代码，而是：

```
运行报错如下：
[粘贴报错信息]

请只修改出错的部分，不要改动其他函数。
分析：报错原因是什么？对应代码的哪一行？怎么修改？
```

如果结果数值不对：

```
程序运行成功，但结果 [X] 与参考值 [Y] 差距过大。

请检查以下几点（逐一回答，不要直接改代码）：
1. 节点和权重的定义是否正确？
2. 从标准区间到 [a,b] 的映射公式是否正确？
3. 误差估计的公式和因子是否正确？
4. 终止条件的判断逻辑是否正确？
```

## 六、考试现场检查清单

拿到代码后，自己肉眼过一遍：

- [ ] 节点/权重是**列表**，不是单个数字
- [ ] 递归函数有 `if ... return` 的终止条件
- [ ] 终止条件在递归调用**之前**
- [ ] 函数内部没有定义另一个函数（嵌套函数是危险信号）
- [ ] 映射公式中有 `(b-a)/2` 的缩放因子
- [ ] 验证代码能打印出误差并与目标精度比较
