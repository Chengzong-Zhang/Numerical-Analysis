===== 第一部分：Cooley-Tukey 基-2 FFT 算法验证 =====

FFT 算法验证：
  测试信号长度 N = 64（= 2^6）
  自实现 FFT 与 MATLAB 内置 fft 的最大误差 = 1.0658e-14
  验证通过：误差在数值精度范围内（< 1e-10）

运算速度对比（DFT vs FFT）：
  信号长度 N = 256：
  直接 DFT 耗时 = 0.0175 秒（O(N^2) = O(65536)）
  自实现 FFT 耗时 = 0.024748 秒（O(N·log₂N) = O(2048)）
  加速比 = 0.7 倍

===== 第二部分：音频信号的 FFT 频域压缩 =====

合成音频信号：
  采样频率 Fs = 1000 Hz
  信号长度 N = 1000 点（时长 1.0 秒）
  成分：50 Hz（幅度2.0）+ 120 Hz（幅度1.0）+ 200 Hz（幅度0.5）+ 噪声

信号补零：1000 → 1024 点（= 2^10，满足基-2 FFT 要求）

频谱分析（前5个主要频率分量）：
  第1大：f =  49.80 Hz, 幅度 = 941.20
  第2大：f = 120.12 Hz, 幅度 = 484.57
  第3大：f =  50.78 Hz, 幅度 = 246.34
  第4大：f = 200.20 Hz, 幅度 = 237.16
  第5大：f =  48.83 Hz, 幅度 = 151.66

===== 频域截断压缩实验 =====

保留比例  20.0%（K =  205 个系数）：RMS = 0.1641, SNR = 19.9 dB
保留比例   5.0%（K =   51 个系数）：RMS = 0.2273, SNR = 17.1 dB
保留比例   1.0%（K =   10 个系数）：RMS = 0.3903, SNR = 12.4 dB

===== 第三部分：图像的 FFT 压缩（示例）=====

合成测试图像：64 × 64 像素
图像 FFT 压缩（保留不同比例频率系数）：
  保留  10.0%（410个系数）：SNR = 24.6 dB
